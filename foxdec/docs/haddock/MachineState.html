Description

These functions are defined using the State (Pred,VCS) monad. Both the read- and write function may update the current predicate, as well as introduce new verification conditions.

invalid_bottom_pointer :: Context -> SimpleExpr -> Bool #

Returns true if a pointer is not suitable for writing to memory. This may happen if the symbolic expression provides no information, i.e., it has Bottom without known pointerbases, and without any sources.

address_is_unwritable :: Context -> SimpleExpr -> Bool #

An address is considered "unwritable" only if it is an immediate address that belongs to a section that is considered unwritable according to Conventions (see section_is_unwritable)
